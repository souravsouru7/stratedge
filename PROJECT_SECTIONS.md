# Stratedge (Edgecipline) — Project Sections

A complete map of every functional area in this codebase.

---

## 1. Authentication

**What it does:** User registration, login, Firebase + JWT flow, OTP verification, password reset, terms acceptance.

| Layer | Files |
|---|---|
| Backend routes | `backend/routes/authRoutes.js` |
| Controller | `backend/controllers/authController.js` |
| Model | `backend/models/Users.js` |
| Frontend pages | `frontend/app/login/`, `register/`, `verify-otp/`, `forgot-password/`, `reset-password/`, `accept-terms/` |
| Frontend hooks | `frontend/features/auth/hooks/` |

---

## 2. Trade Upload & OCR Extraction

**What it does:** User uploads a trade screenshot → saved to Cloudinary → BullMQ job enqueued → OCR worker extracts data using Google Vision / Gemini AI → trade record created.

| Layer | Files |
|---|---|
| Backend routes | `backend/routes/uploadRoutes.js`, `tradeStatusRoutes.js` |
| Controller | `backend/controllers/uploadController.js` |
| Worker | `backend/workers/ocrWorker.js` |
| Services | `backend/services/ocrService.js`, `visionOcrService.js`, `aiExtractionService.js`, `geminiService.js`, `parsingService.js`, `tradeProcessingService.js`, `extractionQualityService.js`, `upload.service.js` |
| Model | `backend/models/ExtractionLog.js` |
| Frontend pages | `frontend/app/upload-trade/`, `upload/` |
| Frontend hooks | `frontend/features/trade/hooks/useUploadTrade.js` |
